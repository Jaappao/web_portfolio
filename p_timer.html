飴と鞭タイマー

途中でお菓子を補給してくれるタイマー



term: 2019.9 - 2019.12
created at: Edison Project
exibition: GUGEN 2019 [GUGEN HP]
what is used:
Language Arduino
Microcomputer M5Stack
Video Editor Adobe Premiere Rush

# What's this?

机に座ったけれどもついついスマホを触りすぎてしまったりする経験、机の上にただ広げてあるだけのお菓子は永遠に食べてしまう経験、ありますよね。このタイマーはまず、あなたのスマホを拘束します。スマホが挿入されていない限りタイマーは動きません。さらに、自分が設定した間隔でお菓子をくれます。飴(お菓子)を与え、鞭(スマホを拘束)を打つ。より生産性を向上させることを考えたタイマーです。

間隔は15分、30分, 60分の3つから選択可能。


# Target User

  • ついついスマホでSNSばかり見続けてしまう学生
  • 机に向かいはするものの、どこか集中力が続かない社会人

# Development Process

3人グループで作成。自分が主に担当したのはマイコンプログラミングと応募用映像制作。

こういった物づくり自体、この飴と鞭タイマーが初めてだった。

アイデア出し・実際の作成の仕方・グループでの分担など本当に不慣れなことばかりだった。結局応募締め切りの一週間前とかに仕様変更が重なってバタバタの応募となった。できなかった機能もあって悔しかったけど、このプロダクトを作る中で、「アイデアを形にすることの楽しさをここで味わえたこと」がすごく自分にとって重要な意味を持っている作品。

マイコン自体を触るのが初めてで非常に苦労した。サーボモータの動作角度をどうやって制御したら確実にお菓子を一つ出せるのか、スマホをどのようにして検知するのかについての実装で悩んだ。スマホの検知について、最初は光センサを用いてスマホの格納状態を検知する予定だったが誤動作が絶えなかったので、制作終盤でスイッチによる検知に切り替えた。検知方法の切り替えは、メソッドを切り替えるだけで実装可能なように設計したので、タイマー動作部に手を加えることなく検知方法を変更できた。
画面表示部について、開発初期はただ単に数字が出てくるだけのつまらない画面だったが、タイマーがこの作品の核をなしている以上、タイマーとしての完成度を上げる必要があった。色を使用したり、図形を使用したり、奥行き感が出るような表示を作ったり、画面下部にはボタンの説明を表示させたりするようにできる範囲で改良を加えた。こういったGUIを設計する際にはなるべく用意されている汎用性の高いライブラリを使用した方がいいということを感じた。座標をいじりまくるのがすごく大変だった。

本当ならばお菓子はなんでも入るようにしたかったが、それは流石に無理があった。モーターを動作させれば出てくるお菓子で、タイマー自体の大きさが勉強机に収まるようなサイズの物を選ぶことにしたために、明治さんのチョコレートになった。また、カートリッジ自体もだいぶ入れやすくなったが、もっと簡単にしたい。というか、そもそも、勉強中にお菓子食べながらやる物なのか。お菓子が定期的に出てくるならその時に集中力散っちゃうじゃないかっていうご意見もFeedbackでいただいた。確かにそう思う人もいるよなと思った。お菓子が出てきた時に休憩モードに移行するようにすればいいかもしれない。次はそういう風な物を作ってみよう。
